# 机器学习课程任务记录

本仓库用于整理和记录机器学习课程中的实验任务、代码实现和实验结果，按实验分目录保存，方便回看、提交和复现。

## 仓库内容

- `实验一 贝叶斯分类器设计与实现/`：实验一相关代码、数据准备、说明文档和结果文件。
- `实验二 决策树算法的设计与实现/`：实验二相关代码、说明文档和结果文件。

## 目录约定

通常每个实验目录会按下面的方式组织：

- `src/`：实验代码。
- `docs/`：实验说明、分析过程、报告草稿等文档。
- `data/`：实验所需的数据集、词表或其他输入文件。
- `results/`：运行后生成的图表、CSV、日志或其他输出文件。
- `requirements.txt`：如需要，可在实验目录内记录对应依赖。

## 使用方式

在仓库根目录下进入对应实验目录，根据实验说明运行代码即可。当前已知的入口如下：

```powershell
python ".\实验一 贝叶斯分类器设计与实现\src\bayes_classifier_experiment.py"
python ".\实验二 决策树算法的设计与实现\src\decision_tree_experiment.py"
```

实验一需要的原始数据与停用词表，默认放在：

```text
实验一 贝叶斯分类器设计与实现\data\
```

如果你使用了其他路径，请以实验目录中的说明为准。

## Commit 规范

提交信息建议使用类似下面的格式：

```text
<type>: <简要说明>
```

常用的 `type` 可以写成：

- `feat`：新增功能、实验内容或代码实现。
- `fix`：修复错误或异常。
- `docs`：只修改文档。
- `refactor`：重构代码，但不改变主要功能。
- `test`：新增或调整测试。
- `chore`：杂项维护、配置整理、依赖调整等。

示例：

- `feat: 完成贝叶斯分类器实验`
- `docs: 更新README说明`
- `fix: 修正决策树输出结果`
